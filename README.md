KingsHandsProject

Тестовое WPF-приложение на C# для просмотра покерных логов из JSON-файлов.

Позволяет выбрать папку, просканировать все .json файлы (включая вложенные) и посмотреть раздачи в удобном виде.

Что умеет
выбор папки через UI
рекурсивный поиск .json файлов
парсинг раздач
отображение:
слева — список столов (TableName)
по центру — список раздач (HandId)
справа — детали (Players, Winners, WinAmount)
корректно обрабатывает битые файлы (не падает)
Как запустить
Через Visual Studio
Открыть KingsHandsProject.csproj
Нажать Run
Через CMake
cmake -B build -G "Visual Studio 17 2022"
cmake --build build --config Release
Тестовые данные

В проекте есть папка:

test-data/

Там лежат:

нормальные JSON-файлы
вложенные папки
битые и частично валидные файлы

Для проверки рабобты можно выбрать эту папку в приложении и нажми “Начать сканирование”.

Немного про реализацию
Архитектура

Используется MVVM:

Views — только XAML
ViewModels — логика UI
Services — работа с файлами и JSON
Models — данные
Потоки

По заданию используется только Thread:

сканирование запускается в отдельном потоке
UI не блокируется
никаких async/await и Task
Обновление UI

Все обновления UI идут через Dispatcher

ObservableCollection не трогается из фонового потока
данные маршалятся в UI-поток
Обработка ошибок
битые JSON-файлы просто пропускаются
приложение продолжает работать
ошибки пишутся в Debug
Особенности
WinAmount хранится как строка (из-за формата вроде "1 000,00$")
невалидные записи внутри файла игнорируются
одинаковые HandId допускаются
